# Contributing to the Sovereign Systems Specification

Thank you for helping define the infrastructure patterns for local-first, sovereign AI. We welcome contributions from systems architects, security engineers, and developers building in this space.

## How to Contribute

### 1. Proposing Terminology or Pattern Updates
If you want to suggest a new term, architectural pattern, or refine an existing definition:
- Check the open Issues to ensure the concept isn't already under discussion.
- Open an Issue using the "New Term Proposal" format, detailing the technical definition, the vector category, and the architectural problem it solves.

### 2. Pull Requests (PRs)
- Fork the repository and create your branch from `main`.
- Ensure your definitions maintain strict technical neutrality—focus on deterministic systems design, resource metrics, and engineering constraints rather than specific vendor software.
- Keep descriptions clear, concise, and focused on production implementation.
- By contributing, you agree that your submissions will be licensed under the project's MIT License.

## Architectural Scope
We explicitly focus on:
- **Local-First Architectures:** Edge execution, local data structures, and deterministic privacy.
- **Resource Optimization:** Eliminating token waste ("Prose Tax") and optimizing context topologies.
- **Data Integrity:** Cryptographic provenance, validation chains, and append-only ledgers.
