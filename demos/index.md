# Why Demos?

Architectural concepts are useful. Working systems are memorable. The purpose of these demonstrations is to showcase exactly how data provenance, custodial bounds, outbound governance, and cryptographic audit trails interact to create trustworthy institutional structures.

```text
Theory ──> Architecture ──> Implementation ──> Experience
```

## Currently Available
### Sovereign Memory Demo
*Status:* Released (v1.0.0)
The flagship reference implementation of the Sovereign Systems Specification. Demonstrates audited institutional memory retrieval with local, tamper-evident transactional ledger custody.
• *Core Features:* Async context offloading, fail-fast startup boundaries, eager relationship hydration, and isolated production CORS mapping.
• *Repository:* [https://github.com/kenwalger/sovereign-memory-demo](https://github.com/kenwalger/sovereign-memory-demo)
• *Screenshots / Video Walkthrough:* Pending production deployment build.
