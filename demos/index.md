# Why Demos?

Architectural concepts are useful. Working systems are memorable. The purpose of these demonstrations is to showcase exactly how data provenance, custodial bounds, outbound governance, and cryptographic audit trails interact to create trustworthy institutional structures.

```text
Theory тФАтФА> Architecture тФАтФА> Implementation тФАтФА> Experience
```
